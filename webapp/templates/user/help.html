Stories and Category

Stories and categories are used to organize your business idea.

Conversations

A Conversation consist of multiple statements and responses. We can add intent to each conversation.

Intents

Intents are used for extracting different entities from the statement. Entity extraction works only if a intent is assigned to the that conversation. Intents name should only contain _, a-z, A-Z, 0-9 characters and any other will automatically erased.

Entity in Statements

We can set words in a statements as a ENTITY using the syntax:

|word|~ENTITY~

word should only contain _, a-z, A-Z, 0-9, space, ,, ., ' characters
ENTITY should only contain _, A-Z characters

Example:

How are you |Aravind|~NAME~?

Entity & Values in Response

ENTITY obtained form the statement and value is obtained from api if given. We can include ENTITY and value in the response using the syntax:

~ENTITY~
~value~

ENTITY should only contain _, A-Z characters
value should only contain _, a-z characters

Example:

Hi ~NAME~.
Your email is ~email~.

Images & Videos in Response

We can include images and videos (supports all formats that respective platforms supports) in responses to the user as following:
Note: URLs should start with https.

Example:

<image|https://example.com/example.png>
<video|https://example.com/example.mp4>
